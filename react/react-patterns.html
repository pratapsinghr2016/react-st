Container / Presentational

üßë‚Äçüç≥ Chef / Waiter

Separate logic (data fetching, state) from UI. Chef handles cooking, waiter just presents beautifully.

// Container (Chef) - handles logic
function UserContainer() {
  const [user, setUser] = useState(null);
  
  useEffect(() => {
    fetchUser().then(setUser);
  }, []);
  
  return <UserCard user={user} />;
}

// Presentational (Waiter) - just displays
function UserCard({ user }) {
  return <div>{user?.name}</div>;
}

Custom Hooks

üìú Family Recipes

Extract and reuse stateful logic. Secret recipes that any chef can use ‚Äî write once, cook anywhere.

function useToggle(initial = false) {
  const [state, setState] = useState(initial);
  
  const toggle = () => setState(prev => !prev);
  
  return [state, toggle];
}

// Usage
const [isOpen, toggleOpen] = useToggle();

Higher-Order Component (HOC)

ü•ò Marination

Wrap a component to add extra powers. Plain paneer goes in, marinated paneer comes out enhanced.

function withLoader(Component) {
  return function Wrapped({ isLoading, ...props }) {
    if (isLoading) return <Spinner />;
    return <Component {...props} />;
  };
}

// Usage
const UserListWithLoader = withLoader(UserList);

Render Props

üç≥ Live Cooking Station

Pass a function that controls what gets rendered. Customer tells chef exactly how they want it cooked.

// Component provides data, you decide rendering

  <p>Cursor at: {position.x}, {position.y}</p>
)} />

// Also works with children as function
<DataFetcher>
  {(data) => <CustomUI data={data} />}
</DataFetcher>

Provider Pattern

üåô Theme Night

Share global state without prop drilling. Monday is Italian night ‚Äî wrap restaurant in Italian context, every dish knows the theme.

const ThemeContext = createContext();

// Wrap app with provider
<ThemeContext.Provider value="dark">
  <App />
</ThemeContext.Provider>

// Any child can consume
function Button() {
  const theme = useContext(ThemeContext);
  return <button className={theme}>Click</button>;
}

Memoization

üî™ Prep Kitchen

Don't recalculate unless inputs change. Prep onions once in the morning, reuse all day. Only re-prep when recipe changes.

// Memoize expensive calculation
const sorted = useMemo(
  () => expensiveSort(items),
  [items]
);

// Memoize callback to prevent child re-renders
const handleClick = useCallback(
  () => doSomething(id),
  [id]
);

// Memoize entire component
const MemoChild = React.memo(ChildComponent);

Lazy Loading

üìÖ Seasonal Menu

Load components only when needed. Don't print entire year's menu ‚Äî show summer items in summer, load winter menu later.

const Dashboard = React.lazy(
  () => import('./Dashboard')
);

function App() {
  return (
    <Suspense fallback={<Spinner />}>
      <Dashboard />
    </Suspense>
  );
}

State Reducer Pattern

üìã Order with Rules

Let consumer customize how state updates. Customer wants to modify order, but kitchen has rules (no extra cheese after 9pm).

function useToggle({ reducer = (s, a) => a.changes }) {
  const [state, setState] = useState({ on: false });
  
  const toggle = () => {
    const changes = { on: !state.on };
    setState(reducer(state, { type: 'toggle', changes }));
  };
  
  return [state.on, toggle];
}

// Consumer controls the rules
useToggle({
  reducer(state, action) {
    if (clickedTooManyTimes) return state; // block change
    return action.changes;
  }
});
