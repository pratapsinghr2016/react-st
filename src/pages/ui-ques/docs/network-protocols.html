1. HTTP Application Layer (L7)

What is it?

HyperText Transfer Protocol - the foundation of data communication on the web. It defines how messages are formatted and transmitted between your browser and web servers.

Purpose & Use Case

Enables fetching web pages. When you type a URL, your browser sends an HTTP request, and the server responds with HTML, CSS, JS, images, etc. It's stateless - each request is independent.

üí° Interview Tip

HTTP is like a conversation where you ask a question (request) and get an answer (response), but the server forgets you after each response.

2. HTTP/3 (QUIC) Application Layer (L7)

What is it?

The latest version of HTTP, built on top of UDP instead of TCP. QUIC stands for Quick UDP Internet Connections.

Purpose & Use Case

Solves the "head-of-line blocking" problem of HTTP/2. Perfect for mobile users who switch between WiFi and cellular, IoT devices, and VR/gaming where low latency matters.

üí° Interview Tip

HTTP/3 is faster because it doesn't need TCP's three-way handshake. Connections are established quicker, and losing one packet doesn't block all streams.

3. HTTPS Application Layer (L7) + TLS

What is it?

HTTP + TLS/SSL encryption. Same as HTTP but with a security layer that encrypts all data.

Purpose & Use Case

Secures sensitive data like passwords, credit cards, and personal info. The padlock icon in your browser means HTTPS is active.

üí° Interview Tip

HTTPS uses asymmetric encryption (public key) for initial handshake, then switches to symmetric encryption (session key) for actual data transfer because it's faster.

4. WebSocket Application Layer (L7)

What is it?

A protocol that provides full-duplex (two-way) communication channels over a single TCP connection.

Purpose & Use Case

Real-time applications like chat apps, live notifications, collaborative editing (Google Docs), stock tickers, multiplayer games. Unlike HTTP, the server can push data to the client without being asked.

üí° Interview Tip

WebSocket starts with an HTTP handshake (upgrade request), then switches to WebSocket protocol. Once connected, both sides can send messages anytime - no more request-response pattern.

5. TCP Transport Layer (L4)

What is it?

Transmission Control Protocol - a connection-oriented protocol that ensures reliable, ordered delivery of data between applications.

Purpose & Use Case

Used when you need guaranteed delivery - web browsing, email, file transfers. It establishes connection via three-way handshake (SYN ‚Üí SYN+ACK ‚Üí ACK).

üí° Interview Tip

TCP is like sending a registered letter - you get confirmation it arrived. It handles packet ordering, retransmission of lost packets, and flow control. The tradeoff is speed for reliability.

6. UDP Transport Layer (L4)

What is it?

User Datagram Protocol - a connectionless protocol that sends data without establishing a connection or guaranteeing delivery.

Purpose & Use Case

Video calls, live streaming, online gaming, DNS lookups - situations where speed matters more than perfect delivery. Missing a frame in a video call is better than waiting for retransmission.

üí° Interview Tip

UDP is like shouting a message - fast, but no guarantee it was heard. No handshake, no acknowledgment, no ordering. You gain speed but lose reliability.

7. SMTP Application Layer (L7)

What is it?

Simple Mail Transfer Protocol - the standard protocol for sending emails across the internet.

Purpose & Use Case

Sending emails from your email client to the mail server, and between mail servers. Note: SMTP is only for sending - receiving uses POP3 or IMAP.

üí° Interview Tip

Email journey: Your Gmail uses SMTP to send ‚Üí Google's SMTP server ‚Üí recipient's SMTP server ‚Üí recipient uses IMAP/POP3 to download. SMTP works like the postal service routing system.

8. FTP Application Layer (L7)

What is it?

File Transfer Protocol - a protocol specifically designed for transferring files between computers on a network.

Purpose & Use Case

Uploading files to web servers, bulk file transfers, maintaining websites. Many developers use FTP/SFTP to deploy websites.

üí° Interview Tip

FTP uses two separate channels - control channel (port 21) for commands like login, list files, etc., and data channel (port 20) for actual file transfer. SFTP is the secure version using SSH.

üéØ Common Interview Follow-ups

"TCP vs UDP - when would you use each?"

TCP for data integrity (file downloads, APIs), UDP for speed-critical apps (video calls, gaming).

"Why did HTTP/3 switch to UDP?"

To avoid TCP's head-of-line blocking where one lost packet stalls everything. QUIC handles reliability at the application level instead.

"How is WebSocket different from HTTP polling?"

HTTP polling repeatedly asks "any updates?" wasting resources. WebSocket keeps one connection open and the server pushes updates instantly.
