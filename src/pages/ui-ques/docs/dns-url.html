üéØ Interview Deep-Dive Questions

Q: Why is DNS caching important?
DNS lookups are expensive (network round trips). Caching at multiple levels (browser, OS, router, ISP) reduces latency and load on DNS servers. This is why the first visit to a site is slower than subsequent visits.
Q: What is the difference between HTTP and HTTPS?
HTTPS adds a TLS/SSL handshake after TCP connection, establishing an encrypted channel. This adds one more round-trip but ensures data security.
Q: What is TCP Keep-Alive and why is it used?
Keep-Alive allows multiple HTTP requests over a single TCP connection, avoiding the overhead of repeated 3-way handshakes. HTTP/1.1 enables this by default. HTTP/2 takes it further with multiplexing.
Q: What happens if DNS resolution fails?
Browser shows "DNS_PROBE_FINISHED_NXDOMAIN" or similar error. Common causes: domain doesn't exist, DNS server is down, or network issues. Flushing DNS cache or changing DNS server often helps.
Q: How does HTTP/2 improve over HTTP/1.1?
HTTP/2 introduces multiplexing (multiple requests in parallel on single connection), header compression, and server push. This eliminates head-of-line blocking at HTTP level.
Q: What is the role of the browser cache vs DNS cache?
DNS cache stores IP address mappings (domain ‚Üí IP). Browser cache stores actual resources (HTML, CSS, JS, images) to avoid re-downloading. Both reduce network requests but serve different purposes.
